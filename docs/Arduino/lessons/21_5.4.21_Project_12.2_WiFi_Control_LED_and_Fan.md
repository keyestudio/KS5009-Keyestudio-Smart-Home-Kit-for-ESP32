### 5.4.21 Progetto 12.2 Controllo WiFi LED e Ventola


#### **1. Descrizione**

In questo progetto impareremo come realizzare diverse funzioni della
casa intelligente accedendo a diverse stringhe sotto l'indirizzo. C'è
uno schermo LCD che può stampare l'indirizzo IP, il che è molto più
comodo.


#### **2. Codice di Test**

⚠️ \ **ATTENZIONE:**\  Dopo aver aperto il file di codice, è necessario
modificare il nome WiFi e le password a cui la scheda di sviluppo ESP32
deve connettersi. Sostituire `ChinaNet-2.4G-0DF0` e `ChinaNet@233` con
il proprio nome WiFi e la propria password rispettivamente. Devi fare
questo prima di caricare il codice; altrimenti la scheda ESP32 non sarà
in grado di connettersi alla rete.

```c
const char* ssid = "ChinaNet-2.4G-0DF0";  // Enter your own WiFi name
const char* password = "ChinaNet@233"; // Enter your own WiFi passwords
```
⚠️ **NOTA: Assicurati che il nome WiFi e le password nel codice siano gli stessi della rete a cui sono connessi il tuo computer, telefono cellulare/tablet, la scheda di sviluppo ESP32 e il router. Devono trovarsi all'interno della stessa rete locale (WiFi).**

⚠️ **NOTA: Il WiFi deve essere su una frequenza di 2.4Ghz; altrimenti, l'ESP32 non può connettersi al WiFi.**

```c
#include <Arduino.h>
#include <WiFi.h>
#include <ESPmDNS.h>
#include <WiFiClient.h>

String item = "0";
const char* ssid = "ChinaNet-2.4G-0DF0";
const char* password = "ChinaNet@233";
WiFiServer server(80);

#include <Wire.h>
#include <LiquidCrystal_I2C.h>
LiquidCrystal_I2C mylcd(0x27,16,2);
//#include <analogWrite.h>
#define fanPin1 19
#define fanPin2 18
#define led_y 12  //Define yellow LED pin as 12

void setup() {
  Serial.begin(115200);
  mylcd.init();
  mylcd.backlight();
  pinMode(led_y, OUTPUT);
  pinMode(fanPin1, OUTPUT);
  pinMode(fanPin2, OUTPUT);

  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("");
  Serial.print("Connected to ");
  Serial.println(ssid);
  Serial.print("IP address: ");
  Serial.println(WiFi.localIP());
  server.begin();
  Serial.println("TCP server started");
  MDNS.addService("http", "tcp", 80);
  mylcd.setCursor(0, 0);
  mylcd.print("ip:");
  mylcd.setCursor(0, 1);
  mylcd.print(WiFi.localIP());  //LCD displays IP address
}

void loop() {
  WiFiClient client = server.available();
  if (!client) {
      return;
  }
  while(client.connected() && !client.available()){
      delay(1);
  }
  String req = client.readStringUntil('\r');
  int addr_start = req.indexOf(' ');
  int addr_end = req.indexOf(' ', addr_start + 1);
  if (addr_start == -1 || addr_end == -1) {
      Serial.print("Invalid request: ");
      Serial.println(req);
      return;
  }
  req = req.substring(addr_start + 1, addr_end);
  item=req;
  Serial.println(item);
  String s;
  if (req == "/")  //Browser can read the information sent by client.println(s) when accessing the address
  {
      IPAddress ip = WiFi.localIP();
      String ipStr = String(ip[0]) + '.' + String(ip[1]) + '.' + String(ip[2]) + '.' + String(ip[3]);
      s = "HTTP/1.1 200 OK\r\nContent-Type: text/html\r\n\r\n<!DOCTYPE HTML>\r\n<html>Hello from ESP32 at ";
      s += ipStr;
      s += "</html>\r\n\r\n";
      Serial.println("Sending 200");
      client.println(s);  //Send the content of string S. When accessing the E-smart home address using a browser, the information can be read.
  }
  if(req == "/led/on") //Browser accesses IP address/led/on
  {
    client.println("turn on the LED");
    digitalWrite(led_y, HIGH);
  }
  if(req == "/led/off") //Browser accesses IP address/led/off
  {
    client.println("turn off the LED");
    digitalWrite(led_y, LOW);
  }
  if(req == "/fan/on") //Browser accesses IP address/fan/on
  {
    client.println("turn on the fan");
    digitalWrite(fanPin1, LOW); //pwm = 0
    analogWrite(fanPin2, 180);
  }
  if(req == "/fan/off") //Browser accesses IP address/fan/on
  {
    client.println("turn off the fan");
    digitalWrite(fanPin1, LOW); //pwm = 0
    analogWrite(fanPin2, 0);
  }
  //client.print(s);
  client.stop();
}
```

#### **3. Risultato del Test**

⚠️ **Nota: Il telefono cellulare o il tablet devono essere connessi alla scheda di sviluppo ESP32 tramite lo stesso WiFi. Altrimenti, non sarà possibile accedere alla pagina di controllo. Inoltre, quando la scheda di sviluppo ESP32 utilizza la funzione WiFi, consuma molta energia. È necessaria un'alimentazione DC esterna per soddisfare il suo fabbisogno energetico per il funzionamento. Se la domanda di potenza non è soddisfatta, la scheda ESP32 continuerà a resettarsi, causando il malfunzionamento del codice.**

Se la casa intelligente è connessa correttamente al WiFi, lo schermo LCD visualizzerà l'indirizzo assegnato.

![immagine61](../media/b61227cbbfd35940c62fac04a680484e.png)

Per accedere all'indirizzo è necessario aggiungere /led/on quando si utilizza il browser, ad esempio il mio indirizzo è 192.168.0.129/led/on. In tal caso le luci LED della casa intelligente saranno accese; se si accede a 192.168.0.129/led/off, allora le luci LED saranno spente.

![immagine62](../media/2788e68263a21922bd1f2178748db72b.png)

Quando il browser accede a 192.168.0.129/fan/on, la ventola della casa intelligente si accenderà e con 192.168.0.129/fan/off si spegnerà.
